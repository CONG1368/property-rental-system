## 1. 系统概述

物业租赁综合管理系统是一款面向物业管理公司的全功能桌面应用，覆盖房源管理、租客签约、收租催缴、合同全生命周期、财务核算等全业务流程。适用于管理 50~500 套房源的中小型物业公司。

### 技术特点

| 特性 | 说明 |
|------|------|
| 运行环境 | Windows 10/11 x64 |
| 数据库 | 内置 SQLite（零配置），可选 MySQL |
| 缓存服务 | 内置 Redis（可选），用于缓存热数据和通知队列 |
| 用户界面 | 现代化桌面应用，Vue3 + Element Plus |
| 数据安全 | 本地存储，JWT 认证 + RBAC 权限控制 |
| 多用户协作 | 9 种角色，模块级权限隔离 |

### 核心功能模块

| 模块 | 功能 |
|------|------|
| 租赁管理 | 房源、租客、账单、催缴、门锁、收租看板、收租日历 |
| 合同管理 | 起草、审批、履行、续约、到期、模板、合规、看板、打印 |
| 财务管理 | 账套、科目、凭证、费用、税务、预算、报表、财务看板 |
| 系统设置 | 用户管理、数据字典、审计日志、打印设置 |

### 定时任务

系统内置以下自动化定时任务：

| 任务 | 执行时间 | 说明 |
|------|---------|------|
| 账单生成 | 每日 02:00 | 为执行中合同自动生成下期账单 |
| 催缴升级 | 每日 08:00 | 逾期账单按规则自动升级催缴等级 |
| 合同到期标记 | 每日 07:00 | 自动标记即将到期/已到期合同 |
| 固定资产折旧 | 每月1日 02:00 | 月度折旧自动计算 |

## 2. 安装与启动

### 2.1 安装方式一：安装包（推荐）

1. 双击运行 `物业租赁综合管理系统-1.0.0-Setup-x64.exe`
2. 选择安装路径（默认 `C:\Program Files\物业租赁综合管理系统`）
3. 点击"安装"，等待完成
4. 桌面会自动创建快捷方式

### 2.2 安装方式二：便携版

1. 解压 `物业租赁综合管理系统-1.0.0-x64.zip` 到任意目录
2. 双击 `物业租赁综合管理系统.exe` 启动

### 2.3 系统要求

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| 操作系统 | Windows 10 64位 | Windows 11 64位 |
| 内存 | 4GB | 8GB 以上 |
| 磁盘空间 | 500MB | 1GB 以上 |
| 其他 | — | 无需安装 Node.js 或数据库 |

## 3. 登录与权限

### 3.1 默认账户

| 用户名 | 密码 | 角色 | 权限范围 |
|--------|------|------|---------|
| admin | admin123 | 管理员 | 全部功能 + 系统设置 |

首次启动时系统自动创建此账户，并初始化标准会计科目表（26个科目）和默认账套。

### 3.2 角色与权限

系统采用基于角色的访问控制（RBAC），共 9 种角色，按模块隔离：

| 角色 | 可访问模块 | 典型职责 |
|------|-----------|---------|
| 管理员 | 全部模块 + 系统设置 | 系统管理、用户维护 |
| 总经理 | 全部业务模块（不含系统设置） | 全局查看、经营决策 |
| 收租主管 | 租赁管理 | 管理收租团队、审批催缴策略 |
| 收租员 | 租赁管理 | 日常收租、账单处理 |
| 财务主管 | 财务管理 | 审核凭证、管理预算 |
| 会计 | 财务管理 | 制作凭证、费用核算 |
| 出纳 | 财务管理 | 收款记录、银行对账 |
| 合同主管 | 合同管理 | 管理合同流程、审批 |
| 法务 | 合同管理 | 合规审查、风险检查 |

> 用户登录后，侧边栏仅显示其角色有权访问的模块，无权模块自动隐藏。

### 3.3 认证机制

- 使用 JWT（JSON Web Token）双令牌认证
- Access Token 有效期 4 小时，Refresh Token 有效期 7 天
- Access Token 过期后自动使用 Refresh Token 续期
- 超过 7 天未登录需重新输入密码

### 3.4 首次登录后建议

1. 修改默认管理员密码（系统设置 → 用户管理 → 点击编辑）
2. 创建各部门用户账号并分配对应角色
3. 检查数据字典配置（系统设置 → 数据字典）
4. 录入房源信息（租赁管理 → 房源管理）

## 5. 租赁管理

> 适用角色：管理员、总经理、收租主管、收租员

### 5.1 房源管理

**路径**：侧边栏 → 租赁管理 → 房源管理

管理所有可出租房源（公寓、厂房、商铺），维护房源基本信息与费用标准。

#### 房源类型

| 类型 | 说明 | 特殊字段 |
|------|------|---------|
| 公寓 | 住宅类 | 楼层、户型 |
| 厂房 | 工业类 | 可租面积（㎡） |
| 商铺 | 商业类 | 临街、楼层 |

#### 房源状态流转

```
空置 → 已预订 → 已出租 → 退租中 → 空置
         ↓                  ↓
       维修中 ←──────────────
```

#### 操作指南

1. **新增房源**：点击"新增房源"按钮
   - 填写名称（如"创新大厦 A 座 401"）
   - 选择类型（公寓/厂房/商铺）
   - 填写面积（㎡）
   - 填写地址
   - 设置费用标准：水费（元/吨）、电费（元/度）、物业费（元/㎡/月）
2. **编辑房源**：列表中点击"编辑"
3. **查看详情**：点击房源行或"详情"按钮，查看完整信息、关联合同、账单历史
4. **筛选**：支持按类型、面积范围筛选

#### 关键字段说明

| 字段 | 说明 |
|------|------|
| 房源编号 | 系统自动生成 |
| 名称 | 房源标识名称 |
| 类型 | 公寓 / 厂房 / 商铺 |
| 面积 | 建筑面积（㎡） |
| 状态 | 空置 / 已预订 / 已出租 / 维修中 / 退租中 |
| 水费标准 | 每吨水单价（元/吨） |
| 电费标准 | 每度电单价（元/度） |
| 物业费标准 | 每㎡每月物业费（元/㎡/月） |

### 5.2 房源批量导入

**路径**：侧边栏 → 租赁管理 → 批量导入

支持通过 Excel 文件批量导入房源数据，适用于初次使用或数据迁移场景。

#### 操作步骤

1. 下载导入模板（系统提供标准模板）
2. 按模板格式填写房源数据
3. 上传 Excel 文件
4. 系统自动校验字段格式和必填项
5. 确认导入，系统逐行写入并反馈导入结果（成功/失败明细）

### 5.3 租客管理

**路径**：侧边栏 → 租赁管理 → 租客管理

管理所有租客（个人和企业），维护联系方式、证件信息、信用评分等。

#### 租客类型

| 类型 | 证件类型 | 必填字段 |
|------|---------|---------|
| 个人 | 身份证 / 护照 | 姓名、手机号、身份证号 |
| 企业 | 营业执照 | 企业名称、统一社会信用代码、联系人 |

#### 信用评分机制

系统根据缴费记录自动计算信用分数（满分 100）：

| 行为 | 影响 |
|------|------|
| 按时缴费 | +5 分/次 |
| 逾期缴费 | -10 分/周 |
| 信用等级 | A(≥80) / B(≥60) / C(≥40) / D(<40) |

#### 操作指南

1. **新增租客**：填写姓名/企业名称、联系电话、证件类型和号码
2. **查看详情**：点击租客行进入详情页，查看个人资料、关联合同、缴费记录、信用历史
3. **关联合同**：租客信息自动关联到其所有合同
4. **筛选搜索**：支持按姓名、手机号搜索，按信用评分筛选

### 5.4 收租管理

**路径**：侧边栏 → 租赁管理 → 收租管理

核心业务模块。管理所有账单的生成、查询、收款操作。每张账单独立拆分各项费用，确保费用透明。

#### 账单生成

系统每日凌晨 2:00 自动根据合同周期生成当期账单。也可手动触发：

1. 点击"生成账单"按钮，系统批量为所有执行中合同生成下一期账单
2. 账单根据合同费用配置自动计算金额

#### 费用构成（每张账单独立拆分）

| 费用项 | 说明 | 金额来源 |
|--------|------|---------|
| 基础租金 | 合同约定月租金 | 合同.rentAmount |
| 水费 | 按房源/合同水费标准 | 合同.waterFee |
| 电费 | 按房源/合同电费标准 | 合同.electricFee |
| 物业费 | 按面积 × 物业费标准 | 合同.propertyFee |
| 其他费用 | 自定义额外费用 | 手动填入 |
| 滞纳金 | 逾期自动计算 | 逾期天数 × 费率 |

> 水费和电费始终独立显示，不会合并为"水电费"。

#### 手动创建账单

1. 点击"手动创建"按钮
2. 选择合同（下拉搜索）
3. 选择账期月份
4. 设置到期日（默认当月 5 日）
5. 分别填写租金、水费、电费、物业费、其他费用
6. 系统自动计算合计金额
7. 确认创建

#### 收款操作

1. 在账单列表中找到待收账单
2. 点击"收款"按钮
3. 输入收款金额（支持部分收款）
4. 选择收款渠道：银行转账 / 微信 / 支付宝 / 现金 / POS / 支票
5. 可填写交易号（选填）
6. 确认收款

收款后系统自动更新账单状态（部分缴 / 已缴），并自动生成记账凭证（借：银行存款 → 贷：各收入科目）。

#### 账单状态

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| 未缴 | 账单已生成，尚未收到付款 | 收款、删除 |
| 部分缴 | 已收到部分款项 | 继续收款 |
| 已缴 | 全部款项已收齐 | 查看详情 |
| 逾期 | 超过到期日仍未缴清 | 收款、催缴 |

#### 批量操作

- 勾选多条未缴账单后可批量删除
- 仅"未缴"状态账单可被删除

#### 账单详情

点击"详情"打开抽屉，展示：
- 账单编号、所属合同、账期、到期日
- 各项费用明细（租金、水费、电费、物业费、其他费用、滞纳金）
- 合计金额、已缴金额、待缴金额
- 收款记录列表（金额、渠道、时间、操作人）

### 5.5 收租日历

**路径**：侧边栏 → 租赁管理 → 收租日历

以日历视图展示每月各日的应收和实收情况，支持按月切换。不同状态用颜色区分（未缴-红色、部分缴-橙色、已缴-绿色），便于按日追踪收款进度。

### 5.6 智能催缴

**路径**：侧边栏 → 租赁管理 → 智能催缴

自动跟踪逾期账单，支持多级催缴升级机制。

#### 催缴等级

| 等级 | 触发条件 | 通知方式 | 处理建议 |
|------|---------|---------|---------|
| 到期提醒 | 到期前 7 天 | 站内信 | 温和提醒 |
| 一级催缴 | 逾期 1-7 天 | 站内信 + 短信 | 温和催缴 |
| 二级催缴 | 逾期 8-30 天 | 站内信 + 短信 + 微信 | 正式催缴通知 |
| 三级催缴 | 逾期 > 30 天 | 全部渠道 + 电话/书面 | 最后通牒，可能启动法律程序 |

#### 操作指南

1. 查看催缴任务列表，支持按等级、合同筛选
2. 对任务执行催缴操作（发送通知/记录电话/生成书面函）
3. 系统记录每次催缴的时间、方式、结果
4. 查看欠款账龄分析（30/60/90/180+ 天分段）

催缴升级由系统每日 8:00 定时自动执行。

### 5.7 收租看板

**路径**：侧边栏 → 租赁管理 → 收租看板

可视化展示收租关键指标，帮助管理者快速掌握整体收租状况。

#### 看板内容

- **KPI 指标**：当月收缴率、逾期率、当月应收/实收金额、欠费户数
- **收租趋势图**：按月展示应收 vs 实收 vs 欠费金额走势
- **逾期率趋势**：按房源类型的月度逾期率变化曲线
- **收款渠道分布**：饼图展示银行/微信/支付宝/现金等渠道占比
- **催缴概览**：各催缴等级任务数量的柱状图

### 5.8 门锁管理

**路径**：侧边栏 → 租赁管理 → 门锁管理

统一管理物业中所有门锁设备，支持智能门锁和传统门锁两种品类，提供密码管理、钥匙流转、操作日志追溯等功能。

#### 门锁品类

| 品类 | 状态字段 | 特殊功能 |
|------|---------|---------|
| 智能门锁 | 在线 / 离线 / 故障 / 未激活 | 远程开锁、临时密码、密码有效期、电量监控、IoT 对接 |
| 传统门锁 | 正常 / 需维修 / 已报废 | 钥匙借出/归还/挂失/作废流转 |

#### 门锁列表页面

1. **统计卡片**：门锁总数、智能门锁数、传统门锁数、异常/低电告警数
2. **筛选条件**：支持按名称/序列号搜索，按品类、房源、状态筛选
3. **动态操作按钮**：智能门锁显示"远程开锁"、"生成临时密码"；传统门锁显示"借出钥匙"
4. **电量/锁芯显示**：智能门锁显示电量进度条（≤30% 红色告警），传统门锁显示锁芯等级

#### 注册门锁

1. 点击"注册门锁"按钮
2. 填写门锁名称
3. 选择品类：智能门锁 / 传统门锁
4. 填写序列号（智能锁必填，传统锁选填）
5. 选择所属房源
6. 填写品牌、型号
7. 选择门锁类型：入户门 / 卧室门 / 办公室门 / 设备间门 / 其他
8. 智能锁额外设置：通讯协议、加密方式、固件版本
9. 传统锁额外设置：锁芯等级（A/B/C/超B级）、原始钥匙数、备用钥匙数

#### 门锁详情页

根据门锁品类动态切换标签页：

**智能门锁标签页**：
- **基本信息**：品牌、型号、序列号、通讯协议、固件版本、所属房源、安装日期
- **运行状态**：在线状态、电量、最后通讯时间、信号强度（Mock 模式下自动模拟）
- **密码管理**：创建/启用/禁用密码，设置密码有效期（起止时间）和使用次数限制，自动记录密码使用日志

**传统门锁标签页**：
- **基本信息**：品牌、型号、锁芯等级、原始钥匙数、备用钥匙数、所属房源
- **钥匙管理**：借出（选择借用人、设置预计归还日期）→ 归还（自动流转回"可借出"）→ 挂失（标记丢失，需确认）→ 作废（彻底报废），完整的钥匙流转记录
- **维修记录**：报修、维修完成、报废流程

**通用标签页（两种品类共用）**：
- **操作日志**：统一记录所有门锁操作（智能锁自动记录 + 传统锁手动登记），含操作类型、操作人、时间、详情

#### 种子数据

系统默认创建 4 套演示门锁（2 智能 + 2 传统），含密码/钥匙/日志演示数据，帮助快速了解门锁管理流程。

### 5.9 打印功能

**路径**：各模块页面操作栏 → 打印按钮

系统内置完善的打印功能，覆盖合同、租客信息、账单、收据等场景，支持系统原生打印和 PDF 导出两种模式。

#### 打印模式

| 模式 | 适用场景 | 说明 |
|------|---------|------|
| 原生打印 | Electron 桌面端 | 调用系统打印对话框，支持选择打印机、纸张、页数 |
| PDF 导出 | 浏览器/桌面端 | 使用 html2canvas + jspdf 生成 PDF 文件下载 |

#### 纸张规格

| 规格 | 尺寸 | 适用文档 |
|------|------|---------|
| A4 竖版 | 210×297mm | 租赁合同、租客信息表、合同批量汇总表 |
| A4 横版 | 297×210mm | 宽幅报表 |
| 80mm 热敏小票 | 80mm×自适应 | 收款收据 |

#### 打印入口

| 页面 | 触发条件 | 输出内容 |
|------|---------|---------|
| 合同详情页 | 状态为执行中/已签订/已到期，点击头部打印按钮 | 完整租赁合同（法律标准格式 + 双方签章位 + 公司 Logo） |
| 租客详情页 | 点击头部打印按钮 | 租客信息表 + 关联合同列表 |
| 收租管理 | 账单列表操作列：已缴→"收据"按钮 / 未缴→"账单"按钮 | 收款收据（80mm 热敏小票）或账单明细（费用分项表 + 金额大写） |
| 合同管理 | 勾选多条合同 → 批量操作栏"打印"按钮 | 合同批量汇总表 |

#### 打印内容样式

- **租赁合同**：法律标准格式，含出租方（甲方）/承租方（乙方）信息、房源信息、合同金额和期限、费用标准、双方签章位、公司 Logo
- **收据**：80mm 热敏小票格式，含公司名称、收据编号、收款金额（大写+小写）、付款人、收款渠道、收款日期、操作员
- **账单明细**：账单编号、费用分项表（租金/水费/电费/物业费/其他费用/滞纳金）、合计金额（大写+小写）、付款信息

## 6. 合同管理

> 适用角色：管理员、总经理、合同主管、法务

### 6.1 合同起草

**路径**：侧边栏 → 合同管理 → 合同管理 → 点击"起草合同"

创建新的租赁合同。支持表单式录入，自动关联房源和租客。

#### 操作步骤

1. **选择模板**（可选）：根据房源类型选择预设合同模板快速填充
2. **填写基本信息**：
   - 合同编号（系统自动生成，可手动修改）
   - 选择房源（下拉搜索）
   - 选择租客（下拉搜索）
   - 月租金、押金金额
   - 计费模式：固定租金 / 阶梯租金 / 抽成模式
   - 合同开始日期、结束日期
   - 付款周期：月付 / 季付 / 年付
3. **配置费用**：
   - 水费（元/月）、电费（元/月）、物业费（元/月）
   - 选择房源后自动带入房源费用标准作为参考
   - 厂房类型可额外配置出租面积
4. **保存**：
   - 点击"保存草稿"：合同状态为"起草中"
   - 点击"提交审批"：合同进入审批流程

### 6.2 合同列表

**路径**：侧边栏 → 合同管理 → 合同管理

查看所有合同的列表视图，支持多维度筛选和批量操作。

#### 合同状态流转

```
起草中 → 审批中 → 已签订 → 执行中 → 到期提醒 → 已到期 → 已终止
              ↓        ↑
           已驳回 ─────┘
```

#### 各状态可用操作

| 状态 | 可用操作 |
|------|---------|
| 起草中 | 编辑、提交审批、删除 |
| 审批中 | 查看进度、驳回、删除 |
| 已驳回 | 编辑（重新提交）、删除 |
| 已签订 | 签署生效、删除 |
| 执行中 | 终止合同、续约、删除 |
| 到期提醒 | 续约、终止、删除 |
| 已到期 | 续约、终止、删除 |
| 已终止 | 删除 |

#### 筛选条件

- 合同状态（支持多选）
- 房源类型
- 到期日期范围（endDateStart / endDateEnd）
- 关键词搜索（合同编号、租客名）

#### 批量操作

- 勾选多条后支持：批量提交审批、批量终止、批量删除
- 批量提交仅对"起草中"和"已驳回"状态有效
- 批量终止仅对"执行中"和"已到期"状态有效

### 6.3 合同审批

**路径**：侧边栏 → 合同管理 → 合同审批

集中处理待审批合同，支持审批通过、驳回及填写审批意见。

#### 操作步骤

1. 查看所有"审批中"状态的合同列表
2. 点击"通过"：合同进入下一审批节点或变为"已签订"
3. 点击"驳回"：填写驳回意见，合同退回至"已驳回"状态
4. 审批操作记录包含：审批人、时间、意见

### 6.4 合同详情

**路径**：合同列表 / 合同看板 → 点击任意合同

#### 展示内容

- **基本信息**：合同编号、金额、日期、付款周期、计费模式、当前状态
- **关联信息**：租客姓名、房源名称和地址
- **费用配置**：
  - 月租金
  - 水费（元/月）、电费（元/月）、物业费（元/月）
  - 月费用合计（租金 + 水 + 电 + 物业）
- **合同附件**：支持上传 PDF / Word / 图片 / Excel 文件（单文件 ≤ 10MB）
- **变更记录**：合同变更历史列表
- **操作日志**：关键操作的时间线记录

### 6.5 合同看板

**路径**：侧边栏 → 合同管理 → 合同看板

以看板（Kanban）方式按状态分组展示合同，8 列对应 8 种状态：

> 起草中 → 审批中 → 已驳回 → 已签订 → 执行中 → 到期提醒 → 已到期 → 已终止

每张卡片显示合同编号、月租金、到期日，点击可进入合同详情。

### 6.6 到期管理

**路径**：侧边栏 → 合同管理 → 到期管理

#### 多周期到期统计

系统提供 6 个周期的到期预览，分阶段管理续约工作：

| 周期 | 颜色标签 | 建议动作 |
|------|---------|---------|
| 已逾期 | 红色 | 紧急处理 |
| 7 天内 | 红色 | 立即续约 |
| 30 天内 | 橙色 | 优先续约 |
| 90 天内 | 蓝色 | 计划续约 |
| 180 天内（半年） | 灰色 | 远期关注 |
| 365 天内（一年） | 绿色 | 信息参考 |

#### 使用方法

1. 在顶部查看 6 周期统计卡片
2. 在"到期日历"下拉框中选择查看月份
3. 表格显示该月到期的所有合同，每行显示剩余天数及颜色标签
4. 点击"续约"按钮进入续约流程

### 6.7 续约管理

**路径**：侧边栏 → 合同管理 → 续约管理

管理即将到期合同的续约评估与操作。合同按剩余天数升序排列（最紧急排最前）。

#### 续约评估

| 评估维度 | 说明 |
|---------|------|
| 信用评分 | 租客历史缴费信用等级 |
| 市场租金 | 周边同类房源租金对比 |
| 违约记录 | 是否有违约/投诉记录 |

#### 续约操作

1. 点击"续约"按钮
2. 设置新合同到期日（默认延后一年）
3. 设置新月租金（默认保持原租金）
4. 填写续约备注（选填）
5. 确认续约，系统自动创建新合同（状态为"起草中"），需重走审批→签署流程

> 续约可从合同列表、到期管理、续约管理三个入口发起。

### 6.8 模板管理

**路径**：侧边栏 → 合同管理 → 模板管理

管理合同模板，按房源类型（公寓/厂房/商铺）分类。起草合同时可选择模板快速填充条款和费用配置。支持创建、编辑、设为默认模板。

### 6.9 合规管理

**路径**：侧边栏 → 合同管理 → 合规管理

对合同进行合规性审查：

- 条款完整性检查（必填条款是否齐全）
- 租金合理性校验（是否偏离市场价过大）
- 租期规范性检查
- 法律风险条款识别
- 生成合规检查报告

## 7. 财务管理

> 适用角色：管理员、总经理、财务主管、会计、出纳

### 7.1 账套管理

**路径**：侧边栏 → 财务管理 → 账套管理

管理多个独立核算的账套。每个账套有独立的会计科目、凭证和报表。

- 系统首次启动自动创建默认账套
- 支持多账套（按项目/公司分别核算）
- 每个账套可设置独立的会计期间

#### 账套信息

| 字段 | 说明 |
|------|------|
| 账套名称 | 如"XX物业2026年度" |
| 公司名称 | 核算主体 |
| 会计期间 | 起止日期 |
| 币种 | 默认 CNY（人民币） |
| 状态 | 启用 / 停用 |

### 7.2 科目管理

**路径**：侧边栏 → 财务管理 → 科目管理

系统内置 26 个标准会计科目，覆盖五大类：

| 类别 | 包含科目 | 方向 |
|------|---------|------|
| 资产类 | 现金、银行存款、应收账款、固定资产等 | 借方 |
| 负债类 | 应付账款、预收租金等 | 贷方 |
| 所有者权益 | 实收资本、未分配利润等 | 贷方 |
| 收入类 | 主营业务收入（含租金/水费/电费/物业费收入） | 贷方 |
| 费用类 | 维修费、办公费、折旧费等 | 借方 |

支持树形层级结构、关键字搜索和按账套筛选。

### 7.3 凭证管理

**路径**：侧边栏 → 财务管理 → 凭证管理

记账凭证的管理核心，支持自动生成和手动录入。

#### 凭证类型

| 类型 | 说明 |
|------|------|
| 收款凭证 | 收到租金、费用等款项 |
| 付款凭证 | 支付维修、办公等费用 |
| 转账凭证 | 内部转账、折旧计提等 |

#### 凭证状态流转

```
草稿 → 待复核 → 待审核 → 已过账
  ↓
已作废
```

#### 自动生成凭证

| 触发条件 | 会计分录 |
|---------|---------|
| 账单收款 | 借：银行存款 → 贷：租金收入/水费收入/电费收入/物业费收入 |
| 费用批准 | 借：费用科目 → 贷：银行存款 |
| 月度折旧 | 借：折旧费 → 贷：累计折旧 |

#### 手动录入凭证

1. 点击"新增凭证"
2. 选择账套、日期、凭证类型
3. 添加分录行（借方/贷方科目、金额、摘要）
4. 确保借贷平衡（合计相等）后保存

#### 凭证编辑

**路径**：凭证管理 → 点击"编辑"进入凭证编辑页

支持多分录行管理，实时显示借贷合计，差额提示。

### 7.4 费用核算

**路径**：侧边栏 → 财务管理 → 费用核算

管理物业运营产生的各类费用。

#### 费用类别

| 类别 | 示例 |
|------|------|
| 维修费 | 设备维修、房屋修缮 |
| 保洁费 | 公共区域清洁 |
| 安保费 | 保安人员、监控维护 |
| 绿化费 | 园林养护 |
| 办公费 | 办公用品、水电 |
| 折旧费 | 固定资产折旧（系统自动计算） |
| 其他 | 其他杂项费用 |

#### 费用分摊

支持配置费用分配规则，将公共费用按比例分摊到各房源：

- **按面积**：按房源面积占比分摊
- **按户数**：平均分摊
- **自定义**：手动指定各房源分摊金额

费用批准后自动生成会计凭证。

### 7.5 税务管理

**路径**：侧边栏 → 财务管理 → 税务管理

#### 税种管理

| 税种 | 计算方式 |

| 房产税 | 从租计征：租金 × 12%；从价计征：房产原值 × 70% × 1.2% |
| 城镇土地使用税 | 面积 × 适用税额 |
| 印花税 | 合同金额 × 税率 |
| 企业所得税 | 应纳税所得额 × 税率 |

#### 操作指南

1. 选择纳税期间
2. 点击"计算"自动生成各税种应纳税额
3. 查看税务报表
4. 导出税务申报数据

### 7.6 预算管理

**路径**：侧边栏 → 财务管理 → 预算管理

#### 预算流程

```
编制中 → 待审核 → 已批准
```

#### 执行监控

- 直观的预算执行进度条

### 7.7 报表中心

**路径**：侧边栏 → 财务管理 → 报表中心

提供多维度财务报表，支持按期间、账套筛选：

| 费用分析表 | 按类别汇总费用支出 |
| 预算执行表 | 预算 vs 实际对比分析 |

### 7.8 财务看板

**路径**：侧边栏 → 财务管理 → 财务看板

可视化展示关键财务指标：

- 收入/费用/利润趋势图
- 费用构成饼图
- 预算执行进度
- 关键财务比率（收缴率、利润率等）

### Q1：如何切换数据库到 MySQL？

在 `backend/.env` 文件中修改配置：

```
DB_DIALECT=mysql
DB_HOST=localhost
DB_PORT=3306
DB_NAME=property_rental
DB_USER=root
DB_PASSWORD=yourpassword
```

修改后重启应用即可。切换到 MySQL 后支持多用户并发访问。

### Q2：数据文件存储在哪里？

**SQLite 模式**（默认）：
- 数据库：`%APPDATA%/物业租赁综合管理系统/data/database.sqlite`
- 上传文件：`%APPDATA%/物业租赁综合管理系统/uploads/`

**MySQL 模式**：
- 数据存储在 MySQL 服务器，应用目录仅存放上传文件

### Q3：如何备份数据？

**SQLite 模式**：直接复制 `database.sqlite` 文件即可。建议设置每日自动备份。

**MySQL 模式**：使用 MySQL 备份工具（如 mysqldump）定期导出。

### Q4：水费和电费在账单中如何查看？

- **列表视图**："费用构成"列显示为"租X + 水X + 电X + 物X"
- **详情抽屉**：分别列出水费和电费金额
- 水费和电费始终独立显示，不会合并

### Q5：收款后凭证在哪里查看？

收款成功后系统自动生成会计凭证，在"财务管理 → 凭证管理"中可查看。凭证类型为"收款凭证"，包含完整的借贷分录。

### Q6：合同到期提醒有哪些周期？

系统提供 6 个周期：
- 已逾期（红色）、7 天内（红色）、30 天内（橙色）、90 天内（蓝色）、180 天内（灰色）、365 天内（绿色）

到期提醒每日 7:00 自动更新。

### Q7：如何续约合同？

三种方式：
1. "续约管理"页面直接点击续约
2. "合同列表"中对执行中/已到期合同点击续约
3. "到期管理"中找到合同后点击续约

续约后自动创建新合同（起草中），需重新走审批→签署流程。

### Q8：新增了一列应当如何刷新数据库？

当代码更新增加了新模型字段后，需要重建数据库：

1. 关闭应用
2. 删除 `backend/data/database.sqlite`（SQLite 模式）
3. 重启应用，系统自动创建新表结构
4. 管理员账户和种子数据会自动重新初始化

> **注意**：此操作会丢失所有数据！仅在新安装或开发测试时使用。

### Q9：应用无法启动怎么办？

1. 检查是否有其他程序占用端口 3001
2. 删除 `%APPDATA%/物业租赁综合管理系统/data/database.sqlite` 后重启（**注意：会丢失所有数据！**）
3. 检查杀毒软件是否拦截了应用
4. 尝试以管理员身份运行
5. 查看 `backend/logs/` 目录下的日志文件排查错误

### Q10：定时任务没有正常执行？

定时任务依赖应用持续运行。如果应用未运行，任务不会补执行。请确保应用处于运行状态，或在任务时间段保持应用开启。

### Q11：不同角色的用户能看到什么区别？

- 侧边栏菜单按角色权限自动过滤（如收租员看不到财务管理菜单）
- 路由守卫阻止无权访问页面的 URL 直接输入
- 仅管理员可访问系统设置模块

### Q12：智能门锁和传统门锁有什么区别？

| 维度 | 智能门锁 | 传统门锁 |
|------|---------|---------|
| 状态跟踪 | 在线/离线/故障/未激活 | 正常/需维修/已报废 |
| 密码管理 | 支持临时密码、有效期、次数限制 | 不支持（使用物理钥匙） |
| 远程控制 | 支持远程开锁 | 不支持 |
| 电量监控 | 实时电量（≤30% 红色告警） | 无 |
| 钥匙管理 | 无（通过密码/远程控制） | 借出→归还→挂失→作废完整流转 |
| IoT 对接 | 支持（Mock 模式默认开启） | 不适用 |

两种品类在同一页面管理，系统根据品类自动切换操作按钮和详情标签页。

### Q13：门锁的 IoT 功能如何启用？

当前门锁 IoT 功能以 Mock 模式运行（自动模拟在线状态、电量、信号强度等），无需配置真实 IoT 平台。需要对接真实平台时，在 `backend/.env` 中配置：

```
DOOR_LOCK_PROVIDER=real
DOOR_LOCK_API_URL=<IoT 平台地址>
DOOR_LOCK_API_KEY=<API 密钥>
```

Mock 模式下的门锁操作会写日志到 `backend/logs/door-lock-provider.jsonl`。

### Q14：打印功能在浏览器中能用吗？

可以。在浏览器环境中，系统自动降级为 PDF 导出模式（html2canvas + jspdf），点击打印按钮直接下载 PDF 文件。在 Electron 桌面端，可以调用系统原生打印对话框选择打印机。

### Q15：打印的合同模板在哪里修改？

- **公司信息**（Logo、签章、公司名称）：在"系统设置 → 打印设置"中配置
- **合同条款内容**：由打印模板 `frontend/src/components/print/ContractPrint.ts` 生成，修改需编辑代码

打印设置中的 Logo 和签章支持实时预览，保存后所有打印文档自动更新。
